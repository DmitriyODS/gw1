# Groove Work — Спецификация системы

Версия: 0.4 · Дата: 2026-03-25

---

## 1. Назначение

Groove Work — внутренняя система управления задачами отдела региональных коммуникаций. Основные сценарии: учёт входящих заявок, трекинг рабочего времени, канбан-доска, аналитика по сотрудникам и отделам.

---

## 2. Роли и права доступа

| Роль | Slug | Описание |
|------|------|----------|
| ТВ-экран | `tv` | Только раздел TV-аналитики. Без авторизации в основной системе. |
| Сотрудник | `staff` | Работает со своими задачами. Таймер. Видит аналитику только по себе. Не может удалять задачи. |
| Руководитель | `manager` | Все задачи. Делегирование. Удаление задач. Полная аналитика. Не управляет пользователями. |
| Администратор | `admin` | Всё, что Руководитель + управление пользователями (≤ manager) и списками. |
| Супер-Админ | `super_admin` | Полный доступ. Управление администраторами. |

**Ограничения создания пользователей:**
- `admin` → может создавать пользователей с ролями: `manager`, `staff`, `tv`
- `super_admin` → может создавать с любой ролью включая `admin`
- Нельзя деактивировать собственный аккаунт

**Валидация учётных данных:**
- Логин: ≥ 4 символа, начинается с буквы `[A-Za-z]`, допустимы `[A-Za-z0-9_-]`
- Пароль: ≥ 6 символов

## 3. Задачи

### 3.1 Статусы
```
new → in_progress ⇄ paused → done → (архив через 7 дней)
```
- Переход в `in_progress` — только через кнопку таймера
- Drag-and-drop на канбане: `new ↔ paused`, нельзя в `in_progress`

### 3.2 Срочность
`slow` (неспешно) < `normal` (обычно) < `important` (важно) < `urgent` (срочно)

### 3.3 Типы задач
Хранятся в таблице `task_types`. Управляются через раздел **Списки** (`/lists`). Новые типы автоматически появляются во всех формах (задачи, планы, ритмы, внешняя форма). Ответственный — `admin` или `super_admin`.

### 3.4 Теги
Предустановленные: `дизайн`, `текст`, `публикация`, `фото/видео`, `внутреннее`, `внешнее`. Назначаются вручную или автоматически по типу задачи.

### 3.5 Вложения
- Максимальный суммарный размер за запрос: **100 МБ**
- При превышении лимита — модальное окно с предложением использовать Яндекс Диск
- Файлы хранятся на диске как UUID-хеши, оригинальные имена — в БД
- Скачивание через Flask с оригинальным именем (требует авторизации)

## 4. Таймер и учёт времени

- Один сотрудник — один активный таймер одновременно
- При старте: задача переходит в `in_progress`, сотрудник становится `assigned_to`
- При паузе: задача → `paused`, `assigned_to_id` сохраняется
- При закрытии: `assigned_to_id = null`
- Время хранится в `time_logs` (started_at, ended_at)
- Делегирование: останавливает таймер текущего владельца

---

## 5. Планы

- Предзаписи задач без статуса и без таймера
- Группируются по `PlanGroup`
- Обязательное поле: **Тип задачи**
- Кнопка «Перенести на доску» создаёт задачу и удаляет план
- Тип `publication` создаёт родительскую задачу + 2 подзадачи

---

## 6. Ритмы

- Шаблоны периодических задач (ежедневно / еженедельно / ежемесячно)
- Запуск: автоматически через daemon или вручную кнопкой
- Обязательное поле: **Тип задачи**
- Хранят `task_type`, `task_tags`, `task_urgency`, `department_id`, `trigger_time`

## 7. Аналитика

### 7.1 Дашборд
- Периоды: день / неделя / месяц / год
- Графики: прогресс (line), по отделам (bar), по типам (horizontal bar)
- Топ по времени и закрытым задачам
- Экспорт: **Excel** (openpyxl) и **PDF** (reportlab + DejaVuSans для кириллицы)

### 7.2 Учёт времени
- Карточки сотрудников с рейтингом
- Рейтинг: `R = 0.5*(N/N_max) + 0.5*(T/T_max) × 100`
  где N — закрытые задачи, T — среднее время на задачу
- Детализация по сотруднику: список задач + статистика по типам

### 7.3 TV-режим
5 слайдов, автосмена 12 сек, обновление данных каждые 30 сек:
1. Сегодня: статусы + типы + время
2. Сегодня: отделы + рейтинг сотрудников
3. Неделя: статусы + типы + время
4. Неделя: отделы + рейтинг
5. Всё время: общая статистика

## 8. Внешняя форма заявок

- URL: `/submit` — без авторизации
- После отправки: перезаполнение формы данными заказчика (prefill в query params)
- Использует актуальный список типов задач из БД

## 9. Списки (справочники)

Раздел `/lists` доступен только `admin` и `super_admin`.

| Справочник | Модель | Поля |
|---|---|---|
| Типы задач | `TaskType` | slug, label, sort_order |

## 10. Яндекс Диск (заготовка)

`app/services/yandex_disk.py` — не активна. При активации:
- Регистрация приложения: https://oauth.yandex.ru
- Переменная окружения: `YANDEX_DISK_TOKEN`
- Зависимость: `pip install yadisk`
- Структура папок: `/GrooveWork/tasks/<task_id>/`
- Методы: `ensure_task_folder()`, `upload_file()`, `upload_task_attachments()`

## 11. Технический стек

## 12. Инфраструктура

### Nginx
- `client_max_body_size 110M`
- Только reverse proxy → Flask (нет статической отдачи `/uploads/`)

### Dockerfile (app/Dockerfile)
- `python:3.11-slim`
- Пакеты: `libpq-dev gcc libjpeg-dev libpng-dev zlib1g-dev fonts-dejavu-core`

### Временная зона
- Все datetime в БД — UTC
- Конвертация в локальное через `TZ_OFFSET_HOURS` (default: 3, МСК)

---

## 13. Безопасность

- CSRF-защита на всех формах (Flask-WTF)
- Файлы отдаются только авторизованным пользователям
- Пароли хранятся через `werkzeug.security.generate_password_hash`
- Роли ограничивают доступ к API-маршрутам на уровне сервера (не только UI)
